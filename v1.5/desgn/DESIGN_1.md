# Jianying-inspired design specification

## Design goal and core task

将本规范用于另一个项目的 Web 工作台、创作工具或 AI 任务入口设计。核心任务是：让用户进入页面后，能快速理解当前能力、输入目标并选择合适的工作模式，随后获得清晰的处理中反馈和可继续编辑的结果。

设计方向提取自：

- `DESIGN-figma.md` 的黑白编辑性框架、语义化 token、胶囊控件和有限的彩色分区。
- Jianying 即梦 AI 当前首页的可观察结构：左侧垂直导航、中央主任务输入区、模式快捷入口、内容发现区和顶部活动提示。

不得复制 Jianying 的品牌、Logo、产品名称、文案、业务内容、素材、图标资源或页面信息架构；只迁移可复用的交互与视觉语言。

## Scope, preservation rules, and evidence limits

### In scope

- 工作台首页或同类 AI 任务入口的布局层级。
- 主输入区、模式选择、快捷能力入口、发现/资源内容区。
- 颜色、字体、间距、圆角、边框、状态和响应式规则。
- 键盘焦点、加载、空状态、错误和减少动效行为。

### Preserve

- 目标项目现有的业务逻辑、数据含义、权限、接口、评分规则和依赖。
- 目标项目已经存在的品牌资产、产品术语和用户任务；参考页只提供设计语言。
- 未被本规范明确覆盖的路由、模块和交互。

### Evidence and limits

- 已观察 Jianying 页面桌面端首页：页面标题为“即梦AI - 一站式AI创作平台”；可见左侧导航、中央 Agent 输入区、能力快捷入口、发现内容区和顶部提示条。
- 页面可见的交互结构包括主输入框、模式选择、技能选择、语音入口、发送按钮、能力卡片、内容标签和轮播前后控制。
- 本规范没有验证登录后的状态、真实生成流程、服务端错误、完整移动端布局、性能指标或目标项目现有代码。
- 参考页的部分活动文案和数据不作为目标项目内容来源。

## Current issues summary

以下不是对目标项目的代码审计，而是将参考页观察转化为设计决策：

| 观察 | 对用户的影响 | 优先级 | 设计方向 |
|---|---|---:|---|
| 主任务集中在页面中央，输入区具有明显视觉重量 | 用户进入后能快速开始，但需要知道输入后会发生什么 | P0 | 保持单一主任务入口；补充输入格式、能力范围和提交后的状态反馈 |
| 多种能力以横向快捷入口并列展示 | 便于发现能力，但过多入口会分散首次任务 | P1 | 首屏最多突出 4–5 个高频入口，其余进入“更多能力”或二级面板 |
| 左侧导航以图标和短标签承担全局导航 | 节省空间，但仅图标容易造成识别负担 | P1 | 桌面端支持图标+标签；折叠后保留可访问名称和 tooltip |
| 发现内容与主创作入口处在同一页面 | 用户可从灵感进入任务，但内容密度可能压低主 CTA | P1 | 主任务区优先完成，再呈现发现内容；移动端将发现内容后置 |
| 顶部活动提示占据首屏横向空间 | 可传达重要运营信息，但不能遮挡任务入口 | P2 | 使用可关闭、可折叠的 announcement；不把运营提示当作主导航 |

## Design principles and explicit exclusions

### Principles

1. **任务先于装饰**：页面第一视觉中心必须是用户当前要完成的任务，而不是品牌口号或运营内容。
2. **轻量的编辑工作台**：以白色或浅色画布、清晰边界和少量黑色重点建立可靠感；用彩色区块标记内容角色，而非铺满背景。
3. **能力可发现但不喧宾夺主**：快捷入口表达能力差异，点击后应给出明确的模式变化，不只切换颜色。
4. **输入、状态、结果连续可见**：用户应能从输入前、处理中、成功和失败状态理解系统当前进度及下一步。
5. **颜色不是唯一状态**：所有状态同时使用文字、图标、结构或 aria 属性表达。
6. **内容优先级稳定**：主任务、模式选择、辅助能力、发现内容按固定优先级重排，不因运营模块随机改变首屏结构。

### Explicit exclusions

- 不使用全局紫蓝渐变、玻璃拟态、发光、装饰性光效或无意义的漂浮背景。
- 不把每个模块都包成统一的浮动圆角卡片。
- 不机械复制 Jianying 的导航名称、能力名称、活动文案或素材。
- 不使用自动播放、持续循环或与任务无关的动效。
- 不用低对比度灰色文字承载关键说明或错误信息。

## Theme and semantic tokens

### Semantic tokens

目标项目应以语义 token 为唯一组件颜色来源；下面的值是默认基线，可按目标项目品牌替换，但需保持角色关系和对比度。

| Token | Default | Usage |
|---|---|---|
| `background` | `#ffffff` | 页面主画布 |
| `foreground` | `#000000` | 主文字、主要图标 |
| `surface` | `#f7f7f5` | 输入区辅助面、次级面板 |
| `surface-strong` | `#ffffff` | 卡片、弹层、输入容器 |
| `primary` | `#000000` | 主按钮、当前关键操作 |
| `primary-foreground` | `#ffffff` | 主按钮文字和图标 |
| `secondary` | `#ffffff` | 次级按钮、未选中控件 |
| `secondary-foreground` | `#000000` | 次级按钮文字 |
| `muted-foreground` | `#777777` | 占位符、辅助说明；不得用于关键内容 |
| `border` | `#e6e6e6` | 输入框、分隔线、卡片边界 |
| `border-soft` | `#f1f1f1` | 非关键分隔 |
| `accent-lime` | `#dceeb1` | 能力或内容分组的柔和强调 |
| `accent-lilac` | `#c5b0f4` | 能力或内容分组的柔和强调 |
| `accent-cream` | `#f4ecd6` | 教程、说明、低风险提示 |
| `accent-mint` | `#c8e6cd` | 成功或已完成内容的背景辅助 |
| `accent-coral` | `#f3c9b6` | 需要注意但非破坏性的提醒 |
| `accent-navy` | `#1f1d3d` | 深色内容区；必须配高对比度文字 |
| `success` | `#1ea64a` | 成功状态，需同时显示成功文本或图标 |
| `destructive` | `#b42318` | 失败、删除或危险操作 |
| `focus-ring` | `#000000` | 键盘焦点环，不能被裁切 |

### Typography

优先使用目标项目已有字体；若无既定字体，采用与 `DESIGN-figma.md` 相近的无衬线正文和等宽辅助文字。中文页面不得依赖英文字体对中文的字形质量。

| Role | Size / line-height | Weight | Usage |
|---|---|---:|---|
| `display` | 48–64px / 1.1 | 500–600 | 页面主标题，桌面端最多一行或两行 |
| `title` | 24–28px / 1.35 | 600 | 模块标题、主任务标题 |
| `body-lg` | 18–20px / 1.45 | 400–500 | 输入提示、核心说明 |
| `body` | 16px / 1.5 | 400 | 正文、按钮、导航标签 |
| `body-sm` | 14px / 1.45 | 400 | 辅助说明、状态文本 |
| `caption` | 12px / 1.3 | 400 | 元信息、时间、非关键标签 |

数字、模型版本或技术参数可使用等宽字体，但不得让大段中文正文使用等宽字体。

### Spacing, shape, and elevation

- 间距基线：`4 / 8 / 12 / 16 / 24 / 32 / 48 / 64 / 96px`。
- 圆角：小控件 `8px`，输入容器和内容卡 `16–24px`，胶囊按钮 `9999px`。
- 边框优先于阴影；默认使用 `1px solid var(--border)`。
- 阴影仅用于弹层、浮层和与画布有明确层级关系的面板，避免卡片普遍悬浮。
- 触控目标最小 `44×44px`；桌面端图标按钮不低于 `40×40px`。

## Information architecture and page priorities

### Desktop order

1. 可关闭的全局提示条（有重要信息时才出现）。
2. 左侧全局导航：当前页面、创建入口、资产/历史、画布或其他项目级模块。
3. 主任务标题：用一句话说明当前工作台能做什么。
4. 主输入区：文本/素材输入、模式、技能、辅助入口和提交操作。
5. 高频能力入口：4–5 个可直接切换的能力卡或模式按钮。
6. 结果或发现区：模板、历史、灵感、资源或最近任务。
7. 帮助入口和次级操作。

### Primary task rules

- 主输入区必须是首屏唯一的主 CTA 区域。
- 进入页面后默认选中一个可用模式，并显示当前模式的名称与用途。
- 能力入口选中后，输入提示、参数面板或结果类型应同步更新。
- 发现内容不得遮挡输入区，也不得将用户强制带离主任务。
- 侧栏折叠、滚动或移动端重排后，用户仍能直接找到创建入口和当前页面位置。

## Component and interaction rules

### Global navigation

- 结构：图标 + 可见标签 + 当前项状态；折叠时保留 `aria-label` 和 tooltip。
- 当前项同时使用背景/边框、文字权重和 `aria-current="page"` 表达。
- 导航项支持 hover、focus、selected、disabled；disabled 不得仅通过变灰表达。
- 移动端改为底部导航或顶部菜单，不压缩成不可点击的小图标列。

### Announcement bar

- 只承载一条短消息和一个可选动作，提供关闭按钮。
- 关闭后在当前会话保持关闭状态；不得自动频繁弹出。
- 文案必须来自目标项目，不复制参考页运营信息。

### Primary composer

- 输入容器由提示区、编辑区、附件/参考入口、模式选择、技能选择、语音或其他辅助入口、提交按钮组成。
- placeholder 说明允许的输入类型和下一步，不承担完整帮助文档。
- 空输入时提交按钮 disabled，并说明需要输入什么；不能只用灰色箭头让用户猜测。
- 聚焦时提升边框或 focus ring，不改变布局尺寸；输入错误时在容器附近提供具体原因。
- 长文本可扩展高度但设置最大高度，超出后在内部滚动；滚动区域必须可键盘访问。
- 上传、拖拽、粘贴等入口应显示文件类型、数量、大小限制和移除操作。

### Mode and skill selector

- 使用胶囊控件表达当前模式，展开面板显示名称、用途、限制和选中状态。
- 选择后保留用户已输入内容；若模式切换会改变参数或消耗，应在操作前说明。
- 菜单支持键盘上下选择、Enter 确认、Escape 关闭，并把焦点返回触发器。
- “自动”类选项必须解释自动决策范围，不得让用户误以为所有参数都不可控。

### Capability card

- 卡片信息顺序：能力名 → 一句用途 → 可选版本/状态 → 进入操作。
- 图像或插图只做识别辅助，不能替代文字名称。
- 选中状态必须有结构变化或文字提示；New、Beta 等标签需要可读文本。
- 卡片数量超过首屏可承载范围时，使用“更多能力”或横向滚动，不强行缩小字体。

### Discovery and content module

- 发现、模板、历史等内容必须标明内容类型、更新时间或来源，避免与生成结果混淆。
- 轮播提供上一个/下一个按钮、当前位置和键盘操作；不依赖自动播放。
- 内容加载失败时保留模块标题并显示重试；无内容时给出解释和可执行的下一步。
- 内容图片使用固定比例容器和明确的替代文本，避免布局跳动。

### Buttons and icon buttons

- 主按钮使用黑底白字，次级按钮使用白底黑字和边框；危险操作使用语义危险色。
- 图标按钮必须有可访问名称、hover tooltip 和 focus ring。
- loading 时保留按钮尺寸，显示进行中状态并阻止重复提交；完成或失败后恢复可操作状态。

## State design

| State | Required behavior |
|---|---|
| Default | 显示当前模式、输入提示和可用操作；主任务不被辅助内容打断 |
| Hover | 只提升可点击性和层级，不改变内容位置；非鼠标用户有等价反馈 |
| Focus | 使用至少 `2px` 的可见 focus ring，不能只依赖颜色或阴影 |
| Selected | 显示名称、图标/结构和选中标识；状态同步到语义属性 |
| Disabled | 说明不可用原因或前置条件；不得只降低透明度 |
| Loading | 保留布局占位，显示阶段性文本或进度；禁用会造成重复提交的操作 |
| Empty | 说明为什么为空，并提供创建、导入或刷新等下一步 |
| Partial | 对已完成和待处理部分分别标识，允许继续或重试 |
| Success | 显示结果、完成状态和下一步操作；提供编辑、复制、保存或继续创作入口 |
| Error | 说明发生了什么、用户能做什么；支持重试，不清空用户输入 |
| Reduced motion | 遵守 `prefers-reduced-motion: reduce`，关闭装饰性位移和自动轮播，仅保留必要状态切换 |

## Responsive and accessibility requirements

### Breakpoints

- `≥1200px`：保留侧栏、中央主任务区和横向能力入口；主内容最大宽度 `1200–1440px`。
- `768–1199px`：侧栏可折叠；能力入口允许横向滚动；主输入区保持完整操作栏。
- `<768px`：改为顶部或底部导航；主输入区单列；发现内容后置；操作按钮保持触控尺寸。
- 页面和组件不得产生横向溢出；横向滚动只用于明确的内容组，并提供滚动提示和键盘焦点。

### Accessibility

- 使用语义化 landmark：`header`、`nav`、`main`、`section`；每个区域有可关联的标题。
- 表单控件必须有 label 或等价可访问名称；placeholder 不替代 label。
- 键盘可以完成导航、模式选择、输入、提交、轮播和关闭操作。
- 文字、图标和边框满足目标项目采用的 WCAG 对比度要求；不能依靠彩色区块单独传达状态。
- 动态状态使用适当的 `aria-live`，避免每次输入都播报；错误信息与控件建立关联。
- 图片有准确替代文本；装饰图片使用空 alt；视频或动画提供暂停/减少动效路径。
- 页面缩放至 `200%` 时，核心任务仍可完成；窄屏和长中文文案不能截断关键操作。

## Acceptance criteria

- Given 用户首次进入工作台，when 页面完成加载，then 用户能在首屏识别当前任务、默认模式、输入位置和提交条件。
- Given 用户键盘操作页面，when 依次经过导航、输入区、模式选择、能力入口和提交按钮，then 焦点顺序符合视觉顺序且每个焦点都清晰可见。
- Given 用户没有输入内容，when 尝试提交，then 提交按钮不可执行并显示具体前置条件。
- Given 用户切换模式，when 选择另一种能力，then 当前输入内容不丢失，模式名称和相关提示同步更新。
- Given 系统正在处理，when 用户查看主任务区，then 能看到处理中状态且不能重复触发同一任务。
- Given 任务失败，when 用户查看错误状态，then 能看到可理解的原因和重试入口，已有输入和素材仍被保留。
- Given 模板、历史或发现内容为空，when 用户打开内容模块，then 显示解释性空状态和下一步，而不是伪造数据。
- Given 页面宽度小于 `768px`，when 用户完成主任务，then 页面无意外横向溢出，输入、模式选择和提交操作仍可用。
- Given 用户开启 reduced motion，when 页面加载或切换状态，then 不播放装饰性动画或自动轮播，必要反馈仍可见。
- Given 目标项目使用参考色彩，when 任一颜色表达状态，then 同时有文字、图标或结构信息，不把颜色作为唯一线索。

## Implementation order for the next AI agent

1. 读取目标项目的 `AGENTS.md`、`MEMORY.md`、`README.md`、现有 `design.md`，确认技术栈和不可变更范围。
2. 将本文件的语义 token 映射到目标项目现有主题系统；不新增依赖，不混用样式方案。
3. 先搭建页面 landmark、侧栏/移动导航和主任务输入区，确保主要任务路径成立。
4. 实现模式选择、技能选择、能力卡和发现内容模块；保留目标项目真实文案和数据。
5. 补齐 default、selected、focus、disabled、loading、empty、success、error 状态。
6. 完成桌面、平板、移动端重排，并检查中文长文案、横向滚动和触控尺寸。
7. 运行项目已有的 lint、test、build 或最小启动检查；用浏览器检查主任务、键盘焦点、移动端溢出和 reduced-motion。
8. 只修改目标项目明确授权的 UI/样式文件；禁止修改业务逻辑、数据含义、权限、后端接口、依赖和参考页内容。
