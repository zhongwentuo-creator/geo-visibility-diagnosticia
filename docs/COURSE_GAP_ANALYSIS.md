## 六、建议学习路径

基于当前项目状态（Python MVP 已跑通），建议按以下优先级补齐：

```
Phase 1（本周）：夯实工程化基础
├── 1.1 添加 pytest 自动化测试（3-4h）
├── 1.2 将 AGENTS.md 改写为 MDC 格式（1-2h）
└── 1.3 添加 FastAPI 包装，本地可 HTTP 调用（2-3h）

Phase 2（下周）：掌握 MCP 核心能力
├── 2.1 学习 MCP 协议，跑通官方示例（2-3h）
├── 2.2 将搜索/豆包 API 封装为 MCP Tool（3-4h）
└── 2.3 在诊断流程中实现 LLM LOOP（4-6h）

Phase 3（第 3 周）：RAG 增强
├── 3.1 接入 ChromaDB，建立品牌知识库（2-3h）
├── 3.2 实现意图识别 + 向量检索（2-3h）
└── 3.3 部署到服务器/Vercel（2-3h）

Phase 4（第 4 周）：LangGraph 自主 Agent
├── 4.1 学习 LangGraph，理解 State/Node/Edge（3-4h）
├── 4.2 将流水线重构为 LangGraph（6-8h）
└── 4.3 实现条件边 + 循环决策（2-3h）
```

**总投入**：约 40-50 小时（4 周，每天 2 小时）

**预期产出**：
- 一个可公网访问的 GEO 诊断服务
- 支持 MCP 工具调用的 Agent 架构
- 基于 LangGraph 的自主决策诊断流程
- 完整的测试覆盖 + 标准 SKILLS 文档

## 七、项目作为"课程练习素材"的适配建议

当前项目非常适合作为 VibeCoding 课程的"课后实战项目"，原因：

| 优势 | 说明 |
|------|------|
| 数据流清晰 | 8 阶段流水线是天然的 LangGraph Node/Edge 映射 |
| 有真实痛点 | JSON 修复、API 降级、并发控制都是真实工程问题 |
| 可渐进增强 | 每学完一天课程，就能对项目做一次升级 |
| 有即时反馈 | 运行脚本就能看到报告，成就感强 |

**建议的课程-项目联动方式**：

```
学完 DAY-01  →  给项目加 Landing Page + 部署
学完 DAY-02  →  给项目接入 MCP + LLM LOOP
学完 DAY-03  →  给项目加 RAG + 测试 + MDC 规范
学完 DAY-04  →  用 LangGraph 重构整个流水线
```
