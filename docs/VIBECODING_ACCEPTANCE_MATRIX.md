## 明确不属于 V1.5 的能力

| 能力 | 归属版本 | 处理原则 |
|---|---|---|
| MCP 工具调用、LLM Loop | V2.0 | V1.5 保持直接 API 调用和固定流水线。 |
| RAG 向量库、意图检索链路 | V2.0 | V1.5 的自然语言字段识别不等同于向量检索。 |
| LangGraph、条件边、循环、自主分工、Harness | V2.0 | V1.5 不宣称具备自主 Agent 能力。 |
| ASR / TTS | 可选 / V2.0 | 当前文本诊断场景不作为 V1.5 发布门禁。 |

## 结项规则

V1.5 只有同时满足以下条件，才可在 `v1.5/docs/plan.md` 标记为“发布验收通过”：

1. Render 生产服务可访问，且生产域名已写入 `CORS_ALLOW_ORIGINS`。
2. 一次真实品牌诊断产生完整 SSE、JSON 和 HTML 报告。
3. 外网桌面端、手机端各完成一次走查，24 小时后复测无阻断问题。
4. URL、测试结果、已知问题和结论回填至 `v1.5/docs/ACCEPTANCE.md`。

## 更新规则

- 只在证据已经产生后更新状态；“代码已写好”不能替代“公网验收通过”。
- 同一事实在本矩阵、`plan.md` 和 `ACCEPTANCE.md` 中使用同一状态与证据链接。
- 新增 V2.0 能力时另建 V2.0 矩阵，不修改本文件的 V1.5 范围。
