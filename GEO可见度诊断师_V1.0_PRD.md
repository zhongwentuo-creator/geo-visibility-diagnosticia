## 一、先讲一个场景

你是「听力熊」儿童智能产品的品牌负责人。有一天，你老板问你：

> **"现在大家都在用豆包、ChatGPT 查东西，你搜搜看，我们的品牌在 AI 里是什么情况？"**

你打开豆包，输入：
- "儿童 AI 学习机哪个牌子好？" → AI 推荐了小度、阿尔法蛋……没有听力熊
- "听力熊怎么样？" → AI 说："这是一个儿童智能产品品牌"——就一句话，没有产品信息、没有购买链接
- "听力熊和小度哪个好？" → AI 详细介绍了小度的功能和价格，提到听力熊时只有一句"也是同类产品"

**你慌了。**

你突然意识到：在 AI 搜索时代，用户不再"搜索网页→点进官网"，而是"直接问 AI→获取总结答案"。如果 AI 不认识你的品牌，或者只说一两句泛泛的话，你就等于在下一个时代的"搜索结果首页"消失了。

但问题是——**你没有一个工具，能系统性地检测品牌在 AI 平台里的表现。**

这就是 **GEO 可见度诊断师** 要解决的问题。

---

## 二、这个产品是什么？

**一句话**：帮你检测品牌在 AI 搜索平台（豆包、ChatGPT、Perplexity 等）中的"存在感"，并告诉你怎么提升。

**三个核心能力**：

### 能力 1：AI 搜索测试

模拟真实用户，在 AI 平台上问 15 个典型问题，记录你的品牌：
- 被提到了几次？
- 是在回答的开头提到（最好的位置），还是末尾一带而过？
- AI 是怎么描述你的？是"推荐购买"还是"不太了解"？
- 竞争对手被提到了吗？说了什么？

### 能力 2：品牌基建扫描

检查你的品牌在互联网上的"基础设施"：
- 官网是否完整？有没有产品介绍、联系方式、用户评价？
- 微信公众号、小红书、抖音、B站……有没有官方账号？有没有在更新内容？
- 36氪、极客公园等科技媒体有没有报道过你？

**为什么重要？** 因为 AI 在回答问题时，优先引用那些"信息完整、来源权威"的品牌内容。如果你的官网是空壳，AI 就"无料可抓"。

### 能力 3：自动出报告 + 优化建议

测试完成后，系统自动生成两份东西：

| 交付物 | 用途 |
|--------|------|
| **一份打分报告** | AIVO 总分（0-100），4 个维度得分，等级评定（优秀/良好/中等/较差/差） |
| **一份优化建议** | 优先级行动清单（高/中/低），Quick Wins（快速见效项），3 个月/6 个月路线图 |

## 三、AIVO 评分是什么？

AIVO = AI 可见度与优化评分。

就像体检报告有血压、血糖、心率各项指标，AIVO 给你的品牌做"AI 体检"，也有 4 项指标：

| 指标 | 含义 | 打个比方 |
|------|------|---------|
| **AI 搜索可见度** | 品牌在 AI 回答中被提到的频率 | 你的品牌上"AI 首页"的次数 |
| **基建完善度** | 官网、自媒体、权威媒体的完整程度 | 你的品牌"信息仓库"建得怎么样 |
| **竞品对比优势** | 和同行业竞品相比，你的相对得分 | 你的"AI 存在感"在行业里排第几 |
| **舆情健康度** | 负面评价占比、风险等级 | 你的品牌"口碑分" |

**四项各占 25% 权重**，算出总分。总分等级：

| 分数 | 等级 | 状态 |
|------|------|------|
| 90-100 | 优秀 | GEO 领导者 |
| 80-89 | 良好 | GEO 先进者 |
| 70-79 | 中等 | GEO 跟随者 |
| 60-69 | 较差 | GEO 滞后者 |
| 0-59 | 差 | GEO 缺失者 |

## 五、真实案例：听力熊

**品牌**：听力熊  
**品类**：儿童 AI 对话智能体  
**诊断平台**：豆包

### 诊断结果

| 维度 | 得分 | 说明 |
|------|------|------|
| AI 搜索可见度 | 58 | 15 问中提及 7 次（47%） |
| 基建完善度 | 55 | 官网几乎空白，内容密度不足 |
| 竞品对比优势 | 60 | 仅为竞品均值的 82% |
| 舆情健康度 | 56 | 负面率 30%，中风险 |
| **AIVO 总分** | **74** | **中等（GEO 跟随者）** |

### 三个最关键的问题

1. **官网是空壳**——AI 搜不到产品信息，只能说"是一个儿童品牌"
2. **竞品被推荐得多**——小度、阿尔法蛋在 AI 回答里出现频率更高
3. **负面舆情影响 AI 判断**——黑猫投诉上有售后问题的帖子，AI 在回答时会引用这些信息

### 优化建议（摘录）

| 优先级 | 行动 | 预期效果 | 时间 |
|--------|------|---------|------|
| 🔴 高 | 重建官网，添加产品详情、用户评价、结构化数据 | +8 分 | 1-2 周 |
| 🔴 高 | 处理黑猫投诉等负面信息，建立售后响应机制 | +5 分 | 1-2 周 |
| 🟡 中 | 入驻小红书/抖音，发布官方内容 | +5 分 | 1 个月内 |
| 🟢 低 | 与 36氪、极客公园等科技媒体合作评测 | +3 分 | 3 个月内 |

**预计 6 个月内可从 74 分提升至 81 分（良好）**。

## 六、和 SEO 有什么区别？

| | SEO（搜索引擎优化） | GEO（生成式引擎优化） |
|--|---------------------|----------------------|
| **优化对象** | 百度、Google 的搜索结果排名 | 豆包、ChatGPT、Perplexity 等 AI 平台的回答内容 |
| **用户行为** | 搜索关键词 → 浏览 10 条结果 → 点击网页 | 直接提问 → 获取 AI 总结答案 |
| **你的目标** | 让自己的网页排在搜索结果前面 | 让自己的品牌被 AI 引用、推荐、详细描述 |
| **优化方法** | 关键词、外链、网页速度 | 结构化数据、权威媒体背书、用户口碑、内容矩阵 |
| **评估工具** | 百度统计、Google Analytics、Ahrefs | **GEO 可见度诊断师** |

**简单说**：SEO 让"用户搜索时能发现你的网页"，GEO 让"用户问 AI 时，AI 能准确推荐你的品牌"。

## 七、这个产品不做什么？

| 不做 | 说明 |
|------|------|
| ❌ 不是实时监测工具 | 诊断是一次性快照，不是 7×24 监控 |
| ❌ 不帮你改网站/发内容 | 只输出诊断和建议，不代执行 |
| ❌ 不替代传统 SEO | 不做百度/Google 关键词排名分析 |
| ❌ 不覆盖付费推广 | 不涉及 SEM、信息流广告投放建议 |
| ❌ 不保证 AI 一定会推荐你 | AI 算法由平台控制，我们只能优化"被引用的概率" |

## 八、V1.0 能做到什么？不能做到什么？

### ✅ 已能做到

- 输入品牌名 + 品类，自动运行完整诊断
- 生成 AIVO 评分（0-100）+ 4 维度详细得分
- 输出 HTML 可视化报告（可直接汇报）
- 输出 JSON 结构化数据（可接入 BI 系统）
- 自动识别 3-5 家竞品并横向对比
- 生成优先级优化建议 + 路线图
- 支持 Kimi、豆包、OpenAI 多个 LLM 后端

### ❌ 还做不到（V2.0 计划）

- 搜索数据是 AI 模拟的，不是真实的搜索 API 数据（V2.0 将接入 SerpAPI）
- 诊断流程是固定的，不能根据结果自动调整策略（V2.0 将引入自主决策 Agent）
- 只能本地运行，没有网页版（V2.0 将部署为 Web 服务）
- 没有自动化测试，偶尔可能出错（V2.0 将补充测试覆盖）

## 九、怎么开始用？

**最快方式**：如果你已经克隆了代码仓库，只需：

```bash
# 1. 配置 API Key（Kimi + 豆包）
cp .env.example .env
# 编辑 .env，填入你的 API Key

# 2. 运行诊断
python main.py --brand "你的品牌名" --category "你的产品品类"

# 3. 查看报告
open output/*/*.html
```

## 十、关于这个项目

**起源**：本项目从 WorkBuddy 平台的"GEO 可见度诊断" Agent 运行记录反推而来。原始 Agent 在 WorkBuddy 中跑通了从品牌信息收集到 HTML 报告生成的完整流程，我们将其逆向工程为可独立运行的 Python 工具。

**构建方式**：使用 **Vibe Coding**（AI 辅助协作编码）——11 个 AI 子代理并行编写 9 个阶段模块 + 2 个工具模块，经过 10 次 Bug 修复联调后，首次测试得分 74/100。

**开源**：MIT License，欢迎提交 Issue 和 PR。
